The 8 Attributes

Pulse.js uses just 8 declarative attributes to handle requests, events, history, and more. Here's a complete overview.

#AttributePurposeRequired
1p-requestHTTP request + response handlingYes
2p-triggerWhen the request firesNo
3p-onLifecycle event hooksNo
4p-historyURL / history managementNo
5p-boostProgressive enhancementNo
6p-oobOut-of-band swap markerNo
7p-ignoreSkip processingNo
8p-inheritInheritance controlNo

1. p-request

The core attribute. Encodes method, URL, headers, body, target, swap behavior, and modifiers into one string.

p-request="{headers} METHOD /url {body} > target.behavior :modifiers"

Examples:

<!-- Simple GET -->
<button p-request="GET /api/data">Load</button>

<!-- POST with body, target, and modifier -->
<button p-request="POST /api/items {this} > #list.append :disable">Add</button>

<!-- DELETE with confirmation -->
<button p-request="DELETE /item/1 > closest tr.outer :confirm('Sure?')">Delete</button>

2. p-trigger

Controls when the request fires. Has smart defaults per element type.

p-trigger="event modifiers"

Defaults: buttons/links → click, forms → submit, inputs → input, selects → change.

<!-- Debounced input -->
<input p-request="GET /search" p-trigger="input debounce 300ms">

<!-- Keyboard shortcut -->
<button p-request="POST /save" p-trigger="keydown[key === 'Enter'] from body">Save</button>

<!-- Polling -->
<div p-request="GET /notifications" p-trigger="every 5s"></div>

3. p-on

Attach handlers to Pulse lifecycle events directly in HTML.

p-on="eventName: handler() | eventName: handler()"
<!-- Log before request -->
<button p-request="GET /data" p-on="pulse:before: console.log('Requesting...')">Load</button>

<!-- Multiple handlers -->
<form p-request="POST /submit {this}"
      p-on="pulse:afterSwap: showToast('Saved!') | pulse:error: showToast('Error')">
</form>

4. p-history

Controls browser history integration.

<!-- Push URL to history -->
<a p-request="GET /page/2 > #content" p-history="push">Page 2</a>

<!-- Replace current URL -->
<button p-request="GET /tab/info > #tab-content" p-history="replace">Info</button>

<!-- Disable history -->
<button p-request="GET /data" p-history="false">Load</button>

5. p-boost

Progressively enhances standard links and forms to use Pulse requests.

<!-- Boost all links in a nav -->
<nav p-boost="true">
  <a href="/home">Home</a>
  <a href="/about">About</a>
  <a href="/contact" p-boost="false">Contact (full reload)</a>
</nav>

<!-- Boost a form -->
<form action="/search" method="GET" p-boost="true">
  <input name="q">
  <button type="submit">Search</button>
</form>

6. p-oob

Marks elements in a server response for out-of-band swaps — updating parts of the page outside the main target.

<!-- Server response with OOB -->
<div id="main-content">Main content here</div>
<div id="notification-count" p-oob="true">3</div>
<div id="sidebar" p-oob="outer">Updated sidebar</div>

7. p-ignore

Prevents Pulse from processing an element and its entire subtree.

<div p-ignore>
  <!-- Pulse will not process any p-* attributes in here -->
  <button p-request="GET /nope">This won't work</button>
</div>

8. p-inherit

Controls how attributes (especially headers) propagate from parent elements.

<!-- Parent sets headers for all children -->
<div p-request="{'X-CSRF': 'token123'}">
  <button p-request="POST /item">Create</button>
  <button p-request="DELETE /item/1">Delete</button>
</div>

<!-- Stop inheritance -->
<div p-inherit="false">
  <button p-request="GET /public">No inherited headers</button>
</div>
